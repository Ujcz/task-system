# 11 交付物矩阵

## 目标

不同任务类型需要不同交付物。本矩阵用于判断“必须交付什么、可以省略什么、需要明确不涉及什么”。

## 交付物矩阵

| 任务类型 | 必须交付 | 可选交付 | 必须明确不涉及 |
| --- | --- | --- | --- |
| 纯后端逻辑小改 | 改动说明、验证说明、风险说明 | 测试用例骨架 | H5/前端、部署、SQL |
| 新增或修改接口 | H5/前端联调文档、测试说明、验证证据 | 产品说明 | 不涉及的端 |
| Admin 配置管理 | Admin 前端联调文档、字段说明、错误码、测试说明 | 诊断文档 | H5、客户端 |
| 数据库改动 | SQL 说明、回滚说明、测试说明、部署说明 | 数据修复方案 | 不涉及的表和旧数据 |
| Redis 改动 | key 规则、TTL、失效策略、回滚/清理说明 | 观测指标 | 不涉及的旧 key |
| MQ 改动 | topic、消息体、生产/消费链路、重试策略、部署说明 | 压测或观测说明 | 不涉及的消费者 |
| 定时任务改动 | 调度规则、主从判断、幂等性、日志、测试说明 | 手动触发说明 | 不涉及的历史任务 |
| 公共契约改动 | 影响模块矩阵、兼容性说明、迁移说明、验证矩阵 | 灰度方案 | 不兼容边界 |
| 配置/字典改动 | 配置 key、默认值、消费方、发布顺序、回滚说明 | 配置校验脚本 | 不涉及的环境 |
| 风险/支付/结算类 | 风险说明、验证证据、回滚方案、观察点 | 发布后监控清单 | 不涉及的资金链路 |

## 硬性规则

- 只要前端可见，就必须有 H5/前端联调说明或明确“不涉及前端”。
- 只要数据库变更，就必须有 SQL/回滚说明。
- 只要配置变更，就必须有配置 key、默认值、消费方和发布顺序。
- 只要公共契约变更，就必须有影响模块矩阵和兼容性说明。
- 只要未跑验证，就必须写未验证项和原因。

## 推荐提示词

```text
请按 .workflow/11-交付物矩阵.md 判断本次任务必须产出哪些交付物。
按“必须产出 / 可选产出 / 明确不涉及”三类输出。
```
