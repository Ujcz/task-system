# 09 角色视角索引

## 目标

同一套交付内容，不同协作方关注点不同。本索引用于快速判断“谁应该看哪份文档、需要什么粒度、哪些内容不该混在一起”。

## 后端开发视角

重点关注：

- TAPD 目标和验收标准。
- 涉及仓库、分支、worktree。
- CodeGraph 索引视角。
- 影响范围和修改点。
- 分层、公共契约、兼容性。
- 验证证据和未验证项。

优先查看：

- `01-TAPD需求读取与澄清.md`
- `02-分析与设计.md`
- `03-工作树与CodeGraph任务环境.md`
- `04-编码实现.md`
- `06-验证与证据.md`

## H5 / 前端视角

重点关注：

- 接口路径、方法、域名和环境。
- 鉴权和公共参数。
- 请求参数。
- 真实 HTTP JSON 返回字段。
- 错误码和前端展示建议。
- 空态、异常态、兼容性。

优先查看：

- `08-H5与前端联调文档.md`
- `模板/H5与前端联调模板.md`

不要把后端诊断接口、Redis 细节、测试清理接口混入前端联调文档，除非它们确实需要前端接入。

## 测试视角

重点关注：

- 测试范围和不涉及范围。
- 测试入口和数据准备。
- 正常场景、边界场景、异常场景。
- 回归范围。
- 已验证项和未验证项。
- 风险提示。

优先查看：

- `06-验证与证据.md`
- `07-交付文档.md`
- `模板/交付检查清单模板.md`

## 产品视角

重点关注：

- 背景和目标。
- 本次解决的问题。
- 用户可感知变化。
- 不变的行为。
- 限制、风险、需要确认的产品语义。

优先查看：

- `01-TAPD需求读取与澄清.md`
- `07-交付文档.md`

产品说明不应堆满类名、方法名、SQL 和 Redis key；必要技术名词要配业务解释。

## 部署 / 运维视角

重点关注：

- 涉及服务和发布顺序。
- 配置、SQL、字典、Redis、MQ 变更。
- 是否需要重启。
- 回滚方案。
- 发布后观察点。
- 日志、指标、告警。

优先查看：

- `07-交付文档.md`
- `11-交付物矩阵.md`
- `12-状态推进规则.md`

## TAPD / 项目管理视角

重点关注：

- 当前状态。
- 主单和子单关系。
- 文档链接。
- 是否可提测。
- 是否有阻塞项。
- 是否需要产品、测试、部署协作。

优先查看：

- `01-TAPD需求读取与澄清.md`
- `10-一页式任务启动模板.md`
- `12-状态推进规则.md`
