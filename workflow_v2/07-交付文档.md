# 07 交付文档

## 目标

把最终结果按读者拆分，让 H5、前端、测试、产品、部署方各自能拿到有用信息，而不是阅读一份混合大文档。

## 给 H5 / 前端

涉及客户端、H5、Admin 前端或任意前端可见接口时，必须单独准备联调文档。不要把接口字段、错误码、交互建议混在测试说明或产品说明里。

```text
H5 / 前端联调说明
- 接入对象: H5 / 客户端 / Admin 前端 / 其他
- 接口清单:
- 环境与域名:
- 鉴权与公共参数:
- 请求方法和路径:
- 请求参数:
- 响应字段:
- 真实 HTTP JSON 示例:
- 错误码和异常文案:
- 前端展示或交互建议:
- 兼容性说明:
- 不涉及接口:
- 联调验收场景:
```

详细规则见 `08-H5与前端联调文档.md`。

## 给测试

```text
测试说明
- 测试范围:
- 不涉及范围:
- 测试入口:
- 数据准备:
- 正常场景:
- 边界场景:
- 异常场景:
- 回归范围:
- 已验证项:
- 未验证项:
- 风险提示:
```

## 给产品

```text
产品说明
- 背景:
- 本次解决的问题:
- 用户可感知变化:
- 不变的行为:
- 配置或开关:
- 限制和风险:
- 需要产品确认的事项:
```

## 给部署

```text
部署说明
- 涉及仓库:
- 涉及服务:
- 发布顺序:
- 配置变更:
- SQL / 字典变更:
- Redis / MQ 变更:
- 是否需要重启:
- 回滚方案:
- 发布后观察点:
- 日志 / 指标 / 告警:
```

## TAPD 回写建议

- 主单评论只放短摘要和文档链接。
- 长文档优先放 Wiki 或本地交付文档。
- 子单更新开发状态和具体进度。
- 不默认覆盖需求正文。
- 写回前确认目标单据、Wiki 路径、标题和摘要。

## 最终交付包

一次完整交付至少应说明：

- 背景和目标。
- 改动范围和不涉及范围。
- 关键改动。
- 影响面。
- 验证证据。
- 未验证项。
- 测试建议。
- H5 / 前端联调说明。
- 产品关注点。
- 部署和回滚说明。
- 残余风险。
