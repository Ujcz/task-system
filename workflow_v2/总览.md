# 个人工作流总览

这个目录用于沉淀个人在 Kissu/Hotchat 多仓 Java 后端任务中的完整工作流。目录已通过根目录 `.gitignore` 忽略，默认不进入业务仓提交范围。

## 工作流目标

把一次任务从“看到 TAPD 原始需求”推进到“可以给 H5、前端、测试、产品、部署协作方交付”的闭环：

```text
TAPD 原始需求
-> 需求读取与澄清
-> 影响分析与任务拆分
-> 设计方案
-> Worktree / CodeGraph 任务环境
-> 编码实现
-> 自审
-> Review
-> 修改 Review 意见
-> 验证与证据整理
-> 给 H5 / 前端 / 测试 / 产品 / 部署的交付文档
-> TAPD 回写 / 提测说明
```

## 核心原则

- `D:\jcz\source` 保持 `master` 主干基线。
- 并行任务放在 `D:\jcz\worktrees\<task-dir>`。
- 主干分析使用 `D:\jcz\source` 的 CodeGraph 索引。
- 任务分支分析使用任务目录自己的 CodeGraph 索引。
- 默认先分析、设计、评审，再进入实现。
- 改公共符号前先做影响分析。
- 代码完成不等于任务完成；交付前必须有证据、风险、未验证项和面向协作方的文档。
- 中文文档、代码、配置、注释默认使用 UTF-8，不加 BOM。

## 推荐使用方式

### 新任务开始

```text
这是 TAPD 驱动的 Kissu 多仓任务。
请按 .workflow/总览.md 的工作流推进，先读取 TAPD 原始需求，产出 Context Pack。
暂时不要改文件。
```

### 一页式任务启动

```text
请按 .workflow/10-一页式任务启动模板.md 收口本次任务启动信息。
先输出一页式任务启动卡，确认 TAPD、仓库、分支、worktree、CodeGraph 索引、风险类型和待确认问题。
```

### 轻量化判断

```text
请先按 .workflow/14-轻量化执行规则.md 判断本次任务应该走快速模式还是完整模式。
如果是快速模式，只保留必要分析、影响确认、改动说明、验证和风险，不强制产出全套文档。
```

### 编码前

```text
请根据 .workflow/02-分析与设计.md 先完成 CodeGraph 影响分析和 Decision Pack。
确认涉及仓库、修改范围、风险、验证计划和交付文档计划后，再进入实现。
```

### 已进入 worktree

```text
当前任务目录是 D:\jcz\worktrees\<task-dir>。
请使用任务级 CodeGraph 索引分析当前分支，不要使用 D:\jcz\source 的 master 视角。
```

### 交付前

```text
请按 .workflow/06-验证与证据.md 和 .workflow/07-交付文档.md 收口。
输出验证证据、未验证项、风险、H5/前端联调说明、测试说明、产品说明和部署说明。
```

### 任务复盘

```text
请按 .workflow/13-任务复盘.md 复盘本次任务。
沉淀卡点、有效提示词、遗漏文档、下次提前确认项和需要更新的工作流规则。
```

## 状态约定

| 状态 | 含义 |
| --- | --- |
| `analysis-ready` | 已读取需求，可以开始分析 |
| `design-ready` | 已形成 Decision Pack，可以确认方案 |
| `code-ready` | 代码层面完成，仍可能缺验证和文档 |
| `review-ready` | 已自审，可进入 Review |
| `test-ready` | 已有提测所需说明和基本证据 |
| `release-ready` | 已具备部署、回滚、观察和风险说明 |
