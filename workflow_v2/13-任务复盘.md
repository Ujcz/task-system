# 13 任务复盘

## 目标

任务结束后沉淀经验，把一次任务中暴露的问题转成下次可复用的规则、模板或提示词。

## 复盘时机

- 提测后。
- 发布后。
- Review 多轮返工后。
- 发现文档遗漏后。
- 发现提示词不好用后。
- 遇到跨仓、配置、部署、数据修复等复杂任务后。

## 复盘内容

```text
任务复盘
- TAPD:
- 任务主题:
- 涉及仓库:
- 最终状态:

做得好的地方:
- 

卡点:
- 

返工原因:
- 

遗漏的文档或信息:
- 

有效提示词:
- 

下次提前确认:
- 

需要更新的工作流规则:
- 
```

## 复盘输出落点

短期可以写在当前任务交付说明中。

长期沉淀时，可以更新：

- `.workflow` 细分文档。
- `.workflow/模板`。
- `AGENTS.md` 中的入口规则。
- 对应 skill 或个人记忆。

## 推荐提示词

```text
请按 .workflow/13-任务复盘.md 复盘本次任务。
重点提炼可复用规则、下次提前确认项、提示词优化和需要补进工作流的文档缺口。
```
