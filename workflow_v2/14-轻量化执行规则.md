# 14 轻量化执行规则

## 目标

工作流是为了减少遗漏，不是为了制造负担。本规则用于判断任务应该走快速模式还是完整模式，让小任务保持轻快，大任务保持可控。

## 模式选择

### 快速模式

适用于：

- 单仓单点小改动。
- 小 bug 修复。
- 仅代码阅读或局部分析。
- 不涉及公共契约。
- 不涉及 H5/前端接口变更。
- 不涉及数据库、Redis、MQ、配置、定时任务。
- 不涉及部署顺序和回滚方案变化。

快速模式最小产出：

```text
快速模式交付
- 任务目标:
- 修改范围:
- 影响确认:
- 改动说明:
- 已验证:
- 未验证:
- 风险:
- 下一步:
```

### 完整模式

适用于：

- TAPD 正式需求或缺陷。
- 多仓联动。
- 公共契约改动。
- H5/前端可见接口改动。
- 数据库、Redis、MQ、配置、定时任务。
- 涉及发布、回滚、灰度、开关或部署顺序。
- 需要给测试、产品、前端或部署方正式交付。

完整模式至少按以下文档判断：

- `01-TAPD需求读取与澄清.md`
- `02-分析与设计.md`
- `03-工作树与CodeGraph任务环境.md`
- `06-验证与证据.md`
- `07-交付文档.md`
- `11-交付物矩阵.md`
- `12-状态推进规则.md`

## 恢复现场模板

并行任务较多时，重新进入任务前先恢复现场：

```text
恢复现场
- 当前任务:
- TAPD:
- 当前目录:
- 当前分支:
- 当前状态:
- 上次做到:
- 下一步:
- 阻塞点:
- 未提交改动:
- 当前应使用的 CodeGraph 索引:
```

## 最小交付口径

任何任务最终至少说明：

- 改了什么。
- 没改什么。
- 怎么验证。
- 没验证什么。
- 有什么风险。
- 下一步给谁。

如果这六点说不清，就不应宣称任务已完成。

## 可以跳过的内容

快速模式下可以跳过：

- 完整 Context Pack。
- 完整 Decision Pack。
- H5/前端联调文档。
- 产品说明。
- 部署说明。
- 任务复盘。

前提是必须明确“不涉及”或“本次无需”，不能沉默省略。

## 不能跳过的内容

以下内容在任何模式都不能跳过：

- 当前任务目标。
- 修改范围。
- 影响范围判断。
- 已验证和未验证项。
- 残余风险。
- 是否涉及 H5/前端、数据库、Redis、MQ、配置、定时任务。

## 推荐提示词

```text
请按 .workflow/14-轻量化执行规则.md 判断本次任务走快速模式还是完整模式。
如果走快速模式，请只输出最小必要交付；如果命中完整模式条件，请说明原因并列出必须产出的文档。
```
